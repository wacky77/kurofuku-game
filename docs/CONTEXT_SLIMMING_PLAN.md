# Claude / Codex コンテキスト軽量化計画

## 現在の状態

- 正式な `CLAUDE.md` は変更しない。
- 軽量版候補は `CLAUDE.slim.proposal.md` として置く。
- この候補ファイルはClaude Codeの予約ファイル名ではないため、自動ロードされない。
- `AGENTS.md` も変更しないため、Codexの既存動作も変わらない。

## Claude側への影響

現段階では影響なし。Claude Codeがプロジェクトメモリとして自動探索するのは主に
`CLAUDE.md` / `CLAUDE.local.md` であり、候補ファイルは必要時に明示して読ませない限り
コンテキストへ入らない。

Claude Codeは `CLAUDE.md` 内の `@path` をimportとして扱う。詳細資料をすべてimportすると
分割後も自動ロードされ、省トークン効果が小さくなる。そのため、本計画では通常のパス表記で
資料の所在だけ案内し、該当作業時に必要箇所だけ読む。

## 安全な移行手順

1. 現行 `CLAUDE.md` を `docs/project/IMPLEMENTATION.md` に複製して完全版を保持する。
2. `CLAUDE.slim.proposal.md` と完全版の必須事項を照合する。
3. Claude Codeの新規セッションで候補版を明示的に読ませ、代表的な質問で理解を確認する。
4. 問題がなければ、現行 `CLAUDE.md` を日付入りバックアップとして残す。
5. 候補版を正式な `CLAUDE.md` に切り替える。
6. Claude Codeの `/memory` で、意図しない詳細資料が自動ロードされていないことを確認する。
7. Codex用 `AGENTS.md` を更新し、常時は軽量版だけ、詳細は必要時だけ読む運用にする。

## ロールバック

切り替え後にClaudeの理解不足が見つかった場合は、正式な `CLAUDE.md` をバックアップ版へ戻す。
コード、アセット、設定ファイルには触れないため、ゲーム本体への影響はない。

## サブエージェント運用案

- 使用する: 独立した複数ファイルの調査、テスト、シミュレーション、比較確認。
- 使用しない: 文言変更、単一ファイルの軽微修正、短い質問への回答。
- 渡す文脈: 対象ファイル、目的、制約、短い返答形式だけ。
- 禁止事項: 全会話履歴や完全版プロジェクト資料を無条件に渡さない。

サブエージェントは時間短縮には有効だが、総トークン量を必ず減らすものではない。
省トークンの中心は、常時ロードするプロジェクトメモリの縮小と必要時読み込みである。
