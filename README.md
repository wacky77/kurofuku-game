# 付け回しマスター

キャバクラの黒服（フロアスタッフ）を題材にした、スマホ向け・一人用のシミュレーションゲーム。
次々来店するお客に対して制限時間内に最適なキャストを「付け回し」、1日の売上目標を目指す。

- **プレイ時間**: 1プレイ 約2〜3分
- **プラットフォーム**: スマホのブラウザ（サーバー不要の静的Webアプリ）
- **技術**: HTML / CSS / Vanilla JavaScript（ビルド不要）

## 遊び方

1. 8人の応募者からキャストを **4人選抜**
2. お客が来店（例：`40代・接待帰り・予算8万円・癒されたい`）
3. 制限時間内にキャスト4人から誰を付けるか選ぶ
4. マッチ度で ★1〜5 判定 → 売上が変動、★4以上でリピーター獲得
5. 1日6組をさばいて売上目標を達成できるか

### ゲーム要素
- **ニーズ判定**: 癒し / トーク / 高単価 / 笑顔 の相性で結果が変わる
- **体力管理**: 接客するとキャストの体力が減り、休ませると回復。連投は成功率ダウン
- **ランダムイベント**: 誕生日客・社長来店・酔っ払い・VIP指名（高単価＆ボーナス倍率）
- **制限時間**: 迷うと機会損失
- **連続的中コンボ**: 最適キャストを選び続けると売上ボーナス（最大 +40%）。外すとリセット
- **リピーター再来店（指名客）**: ★4以上を出すとその客が後日「〇〇ちゃんをご指名」で再来店。応えれば高単価＆常連化、外すと二度と来ない（＝入れ替え判断が効いてくる）
- **キャスト育成**: 接客でEXPが貯まりレベルアップ。レベルにつき売上補正。編成画面の ℹ️ でレベル・EXP進捗・能力・体力を確認できる
- **黒服ランク**: 通算売上で役職が昇格（見習い黒服→黒服→主任→副店長→店長→エリアマネージャー→伝説の黒服）。日次リザルトに次の役職までの進捗、昇格時は演出＋タイトルに最高役職を表示
- **席数拡大**: 黒服の成長で席が増える（DAY4で5席・DAY7で6席）。開店時に新メンバーが自動加入し、扱えるキャストが増える（＝ニーズ網羅は楽に／覚える顔は増える）
- **効果音・BGM**: MP3音源をWeb Audioで再生し、読み込み失敗時は一部を合成音へフォールバック。🔊/🔇 で一括切替（設定は保存）
- **結果シェア**: 通算売上・最大コンボをワンタップで共有（Web Share API／クリップボード）

## 起動方法

静的ファイルなので、フォルダ内でローカルサーバーを立てるだけ。

```bash
cd kurofuku-game
python3 -m http.server 5500
# ブラウザで http://localhost:5500 を開く
```

`index.html` をブラウザで直接開いてもOK。

## ファイル構成

```
kurofuku-game/
├── index.html            # エントリポイント（PWA メタ・SW 登録込み）
├── manifest.webmanifest  # PWA マニフェスト
├── sw.js                 # Service Worker（オフライン対応・アプリシェルキャッシュ）
├── css/style.css         # スタイル（ダーク×ゴールドのキャバ内装イメージ）
├── js/assets.js          # 画像アセットのマニフェストと表示ヘルパー
├── js/data.js            # キャスト・お客・イベント等のゲームデータ
├── js/avatars.js         # SVG 似顔絵／客アイコンの動的生成
├── js/audio.js           # MP3音源＋Web Audio合成フォールバック
├── js/achieve.js         # 実績の判定・保存・通知
├── js/game.js            # ゲームロジック＆画面描画
└── assets/               # 配信用の画像・音声・アイコン
```

> 更新時は `index.html` の `?v=N`、`js/assets.js` の `ASSET_V`、`sw.js` の `CACHE` / `MEDIA_CACHE` / `MEDIA_V` / `ASSETS` の版数を揃えて上げること。Service Workerは配信用画像・音声も事前保存し、旧世代のキャッシュを自動削除する。

## 実装済み
- ハイスコア保存（localStorage・通算売上 Top5）
- 難易度カーブ（DAY 進行で目標↑・制限時間↓・来店数↑・イベント率↑）。目標は `300,000 + (DAY-1)×55,000`。新メカニクス込みでシミュレーション調整済み（序盤易→中盤で腕前が分かれ、雑なプレイは4〜5日で脱落、熟練で長期生存）
- 翌日のメンバー編成（最大2人入替・体力の翌日繰越）
- 連続的中コンボ／リピーター再来店（指名客）／キャスト育成（EXP・レベル＋詳細画面）
- 席数拡大（DAY4で5席・DAY7で6席、開店時に新メンバー自動加入）
- 黒服ランク（通算売上で昇格・最高役職の表示）
- MP3効果音＋ラウンジ風BGM（Web Audio・合成音フォールバック・ミュート切替）
- PWA 化（ホーム画面追加・画像と音声を含むオフラインプレイ）・結果シェア
